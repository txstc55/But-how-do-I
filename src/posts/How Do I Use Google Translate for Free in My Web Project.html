So we all know google has translate service on web. But that's only for web use. If you want to have a service that sends a request to google and have them translate something for you, google also supports paid service here: https://cloud.google.com/translate.

But, but they have the browser based service for free, can't we somehow use that? Well no. End of the story.

Wait, that's not the end. If you do a bit of search, you will find that google has a translation service for internal use, but here's the catch: it only works if the request is sent not from a browser.

End of the story? Not so quite. If you haven't done so, I would suggest you to read this post.

Now, assuming that you have finished reading that post, let's get back to business. You can of course send post requests yourself, which includes finding the correct hash for your message, finding the correct parameters to retrieve etc. But you can also use this package: translate-tools. And let's suppose that you have already set up a CORS-Anywhere. You can now initialize the translator like this: 

import { GoogleTranslator } from "@translate-tools/core/translators/GoogleTranslator";

// initialize the translator
var translator = new GoogleTranslator({
  corsProxy: "YOUR_CORS_ANYWHERE_URL",
});
